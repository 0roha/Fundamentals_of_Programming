Introduction to Node.js

Node.js is a JavaScript runtime built on Chrome's V8 JavaScript engine. It allows you to run JavaScript on the server, enabling full-stack JavaScript development.

Key Features:

  • Non-blocking I/O: Handles multiple operations concurrently
  • Event-driven architecture: Uses events and callbacks for async operations
  • Single-threaded: Uses a single main thread with worker threads for heavy operations
  • NPM ecosystem: Access to thousands of open-source packages
// Simple Node.js example

console.log(message);

Node.js Modules

Node.js uses a module system to organize code into reusable pieces. There are three types of modules:

1. Core Modules

Built-in modules that come with Node.js installation:

  • fs - File system operations
  • http - HTTP server and client
  • path - File path utilities
  • events - Event handling

2. Local Modules

Modules you create in your own files:

// mathUtils.js
const add = (a, b) => a + b;
const multiply = (a, b) => a * b;

module.exports = { add, multiply };
// app.js
const mathUtils = require('./mathUtils');

console.log(mathUtils.add(5, 3)); // Output: 8
console.log(mathUtils.multiply(5, 3)); // Output: 15

3. Third-party Modules

Modules installed via NPM (Node Package Manager):

// Install with: npm install express
const express = require('express');
const app = express();

Event Loop & Asynchronous Programming

Node.js uses an event-driven, non-blocking I/O model that makes it lightweight and efficient.

The Event Loop

The event loop allows Node.js to perform non-blocking I/O operations despite being single-threaded.

// Asynchronous example with callbacks
const fs = require('fs');

console.log('Start');

fs.readFile('example.txt', 'utf8', (err, data) => {
  if (err) throw err;
  console.log('File content:', data);
});

console.log('End');
// Output order: Start, End, File content

Promises and Async/Await

Modern approaches to handle asynchronous operations:

// Using Promises
const fetchData = () => {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      resolve('Data received');
    }, 1000);
  });
};

fetchData().then(data => console.log(data));
// Using Async/Await
const getData = async () => {
  try {
    const data = await fetchData();
    console.log(data);
  } catch (error) {
    console.error(error);
  }
};

getData();

File System Operations

The fs module provides an API for interacting with the file system.

Reading Files

const fs = require('fs');

// Synchronous reading
try {
  const data = fs.readFileSync('file.txt', 'utf8');
  console.log(data);
} catch (err) {
  console.error(err);
}

// Asynchronous reading
fs.readFile('file.txt', 'utf8', (err, data) => {
  if (err) throw err;
  console.log(data);
});

Writing Files

const fs = require('fs');

// Write to a file
fs.writeFile('message.txt', 'Hello Node.js', (err) => {
  if (err) throw err;
  console.log('File has been saved!');
});

// Append to a file
fs.appendFile('message.txt', '\nMore data', (err) => {
  if (err) throw err;
  console.log('Data appended!');
});

Creating an HTTP Server

Node.js makes it easy to create web servers using the http module.

Basic HTTP Server

const http = require('http');

const server = http.createServer((req, res) => {
  // Set response header
  res.writeHead(200, { 'Content-Type': 'text/html' });
  
  // Send response
  res.end('<h1>Hello from Node.js Server!</h1>');
});

// Start server on port 3000
server.listen(3000, () => {
  console.log('Server running at http://localhost:3000/');
});

Handling Routes

const http = require('http');

const server = http.createServer((req, res) => {
  if (req.url === '/') {
    res.writeHead(200, { 'Content-Type': 'text/html' });
    res.end('<h1>Home Page</h1>');
  } else if (req.url === '/about') {
    res.writeHead(200, { 'Content-Type': 'text/html' });
    res.end('<h1>About Page</h1>');
  } else {
    res.writeHead(404, { 'Content-Type': 'text/html' });
    res.end('<h1>404 - Page Not Found</h1>');
  }
});

server.listen(3000);
